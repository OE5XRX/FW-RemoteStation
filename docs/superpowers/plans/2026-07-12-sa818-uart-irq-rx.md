# SA818 IRQ-Driven UART RX Cleanup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refactor the SA818 interrupt-driven AT-command RX path to be spin-safe on real hardware, single-path, and idiomatic, without changing observable behavior.

**Architecture:** Replace the byte-wise `k_msgq` transport with a `struct ring_buf` + binary `k_sem`. Harden the UART ISR against a stuck non-RX pending interrupt. Collapse the read function to one drain-and-wait loop with a single timeout check. Remove the dead poll fallback so the IRQ path is mandatory (init failure is hard).

**Tech Stack:** Zephyr RTOS (`ring_buf`, `k_sem`, UART interrupt-driven API), C++ TU exposing a C ABI, native_sim + Twister + pytest for verification, clang-format-18.

## Global Constraints

- No dynamic allocation; no exceptions; no RTTI. (fixed-size `ring_buf` buffer only)
- Driver result codes are `[[nodiscard]]` C enums (`enum sa818_result`) — do not discard.
- Formatting: clang-format-18, 2-space indent, 160-column limit; must produce no diff.
- Behavior-preserving: `tests/sim_shell` (79 cases) must stay 79/79 green **with no test changes**.
- Firmware stays thin: no persistence/access/platform logic added.
- Ring buffer size: `SA818_AT_RX_RB_SIZE = 256` (> `SA818_AT_RESPONSE_MAX_LEN` = 128).
- Overrun policy: log only (`LOG_WRN`), no new error code.
- Reference spec: `docs/superpowers/specs/2026-07-12-sa818-uart-irq-rx-design.md`.

- [ ] **Step 3: Rewrite the ISR in sa818_at.cpp (spin-safe, ring_buf + sem)**

Replace the entire `sa818_uart_isr` function with:

```c
static void sa818_uart_isr(const struct device *uart, void *user_data) {
  struct sa818_data *data = static_cast<struct sa818_data *>(user_data);
  if (!uart || !data) {
    return;
  }

  while (uart_irq_update(uart) && uart_irq_is_pending(uart)) {
    if (!uart_irq_rx_ready(uart)) {
      break; /* only RX IRQ is enabled; anything else -> leave, do not spin */
    }

    uint8_t buf[16];
    int n = uart_fifo_read(uart, buf, sizeof(buf));
    if (n <= 0) {
      break;
    }
    if (ring_buf_put(&data->at_rx_rb, buf, n) < static_cast<uint32_t>(n)) {
      data->at_rx_overrun = true;
    }
    k_sem_give(&data->at_rx_sem);
  }
}
```

- [ ] **Step 6: Rewrite uart_read_response as a single drain-and-wait loop**

Replace the entire `uart_read_response` function (note the dropped `uart`
parameter) with:

```c
static sa818_result uart_read_response(struct sa818_data *data, char *response, size_t response_len, uint32_t timeout_ms) {
  if (!data || !response || response_len == 0) {
    return SA818_ERROR_INVALID_PARAM;
  }

  memset(response, 0, response_len);
  size_t pos = 0;
  const int64_t start = k_uptime_get();

  while (pos < response_len - 1) {
    uint8_t c;
    while (ring_buf_get(&data->at_rx_rb, &c, 1) == 1) {
      if (c == '\n') {
        response[pos] = '\0';
        return SA818_OK;
      }
      if (c == '\r') {
        continue;
      }
      response[pos++] = c;
      if (pos >= response_len - 1) {
        break;
      }
    }

    int32_t remaining = static_cast<int32_t>(timeout_ms) - static_cast<int32_t>(k_uptime_get() - start);
    if (remaining <= 0) {
      LOG_ERR("UART read timeout");
      return SA818_ERROR_TIMEOUT;
    }
    if (k_sem_take(&data->at_rx_sem, K_MSEC(remaining)) != 0) {
      LOG_ERR("UART read timeout");
      return SA818_ERROR_TIMEOUT;
    }
  }

  response[response_len - 1] = '\0';
  return SA818_OK;
}
```

## Notes on test coverage (no new test task)

The existing `tests/sim_shell` suite is the regression guard and is sufficient
here: the refactor is behavior-preserving by design. A dedicated overrun test is
**not** added because the overrun branch is effectively unreachable through
normal AT traffic — response buffers are ≤ `SA818_AT_RESPONSE_MAX_LEN` (128) and
`uart_read_response` stops at `response_len-1`, so the 256-byte ring cannot fill
from a single bounded response. The overrun flag exists as a safety signal if
that assumption ever changes, and is exercised implicitly by every AT round-trip
(it must remain false). Adding a synthetic overrun test would require injecting
an unbounded byte stream the PTY simulator does not model, so it is deliberately
out of scope.
